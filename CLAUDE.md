# 絶対に守るべきルール

以下のルールを **絶対に** 守ること。

## 1. Plan Mode の使用

コードを書く前に、 **必ず Plan Mode で起動** し、ユーザーとプランを練ること。

実装の方向性をユーザーに確認し、合意を得てから作業を開始すること。

## 2. 音声通知の実行（最優先ルール）

**このルールは Plan Mode の制約よりも優先される。Plan Mode 中であっても音声ツールは必ず実行すること。**

以下のようなユーザーの操作を待つタイミングで **必ず音声を鳴らす** こと:

- AskUserQuestion ツールを使用してユーザーに質問するとき（質問と同時に音声を鳴らす）
- ExitPlanMode ツールを使用してプランを提示するとき（プラン提示と同時に音声を鳴らす）
- Bash などでコマンドの実行許可を得るとき
- コーディングが完了したとき

音声を鳴らすには以下の MCP ツールを使用すること:

| MCP ツール名              | 用途                     |
| :----------------------: | :----------------------: |
| mcp__play-sound__play_glass | ユーザー操作待ちの通知 |

**重要:** 音声ツールは readonly であり、システムを変更しない。Plan Mode 中でも安全に実行できる。
